Similarity Algorithm: Weighted Composite
totalScore = (nameSimilarity × 0.3) + (structureSimilarity × 0.5) + (semanticSimilarity × 0.2)

┌─ Name Similarity (30%) ──────────────────────────────────┐
│  1. Normalize: lowercase, strip underscores/hyphens      │
│  2. Exact match → 100%                                   │
│  3. Synonym map check (users↔user_accounts, etc.) → 90%  │
│  4. Levenshtein distance → scaled 0-100%                 │
│  5. Substring containment bonus (+15%)                    │
└──────────────────────────────────────────────────────────┘

┌─ Structure Similarity (50%) ─────────────────────────────┐
│  1. Column count ratio                                    │
│  2. Column name overlap (Jaccard coefficient)             │
│  3. Column type matching for overlapping columns          │
│  4. Constraint similarity (PK, FK, UNIQUE patterns)       │
│  5. Index similarity                                      │
└──────────────────────────────────────────────────────────┘

┌─ Semantic Similarity (20%) ──────────────────────────────┐
│  1. Foreign key relationship patterns                     │
│  2. Column naming conventions match                       │
│  3. Data type distribution similarity                     │
│  4. Timestamp column patterns (created_at vs created_ts)  │
└──────────────────────────────────────────────────────────┘

Thresholds:
  ≥ 90%  →  "Exact Match" (green)
  ≥ 75%  →  "Likely Similar" (yellow) — flagged for review
  ≥ 50%  →  "Partial Match" (orange)
  < 50%  →  "Different" (no highlight)
